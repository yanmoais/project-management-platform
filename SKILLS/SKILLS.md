# 技能规则文档 (Skills Rules)

## 1. 代码注释规范
- **强制中文注释**：在生成、修改、重写代码时，所有注释必须使用中文。
- **全面覆盖**：函数、类、方法及关键逻辑块均需添加中文注释，描述其功能、参数、返回值及实现逻辑。

## 2. 方法调用与逻辑校验
- **调用一致性**：引入新方法调用时，必须严格检查调用方与被调用方的参数、返回值是否一致。
- **实现验证**：确保被调用的方法已正确生成，且方法体内部逻辑正确无误，不得出现空引用或逻辑漏洞。
- **闭环检查**：在编写调用逻辑的同时，必须同步确认被调用方法的具体实现。

## 3. 数据库操作规范
- **表结构为准**：进行数据库操作前，必须查询数据库表结构和 `models.py` 进行双重 Check，确保字段结构无误。
- **字段映射**：严格按照现有表结构进行字段映射，严禁私自臆造或修改字段名。
- **一致性保证**：代码中的字段命名必须与数据库表字段完全匹配，防止因字段名不一致导致的运行错误。
