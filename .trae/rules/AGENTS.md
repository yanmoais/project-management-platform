## 01. 语言与交互绝对底线 (Language)
- **强制全量中文**：代码的注释 (Docstring)、复杂逻辑行内解释、AI 交互应答、控制台输出，**100% 使用中文**。
- **严禁英文注释**：禁止在代码中夹杂任何英文解释。所有函数/类必须标注中文的：功能描述、参数 (Args) 类型、返回值 (Returns)。
## 02. 架构与全链路规范 (Architecture & UI)
- **核心栈**：Python(FastApi) + Vue3。严禁变更。
- **仓储模式**：业务逻辑【必须】放在 Service 层。前端 API 请求统一存放在 `src/api/`。
- **前端极严规范**：
  - 【强制】所有 UI 交互必须使用 **Element Plus** 组件。
  - 【严禁】使用原生 HTML 标签（如 `<button>`, `<input>`）。
  - 页面新增必同频注册 `src/router/`。
- **后端闭环**：新路由在 `routes/*_routes.py` 编写，且【必须】在 `app.py` 中注册蓝图。
## 03. 数据库与零幻觉 (DB & Zero Hallucination)
- **最高红线 (DDL)**：所有表结构变更【仅限】在 `create.sql` 末尾**追加 (APPEND)**。严禁覆盖、修改或删除历史 SQL 代码！
- **统一配置**：MySQL 账号密码严禁硬编码，必须读取 `src/settings.js`。
- **1:1 绝对映射**：编写 DB 读写逻辑前，必须前置比对 `create.sql` 与 `models.py`。代码字段名必须与表结构 1:1 绝对匹配，**严禁私自臆造、缩写或推测字段名**。
## 04. 质量防错逻辑 (QA & Logic Audit)
- **拒绝空壳**：严禁生成空引用或仅用 `pass` 占位的方法。定义新调用时，【必须】同步完成被调用方的完整内部逻辑实现。
- **签名对齐**：引入新方法时，强制双向检查调用方与被调用方的：参数数量、类型、返回值，确保逻辑闭环无误。
